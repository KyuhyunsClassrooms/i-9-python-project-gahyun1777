# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제: 반감기에 따른 체내 잔류 농도와 용량 계산기

예) 기분과 시간에 따른 활동 추천기

---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.

예) 사용자가 현재 기분과 사용할 수 있는 시간을 입력하면, 조건에 맞는 활동을 추천해 주고 싶다.

내 설명: 사용자가 약물의 이름, 반감기, 최소 유효 농도 등 필요한 정보를 입력하면 몇 시간 후 효과가 사라지는지 알려주고 복용 간격을 추천해준다.


---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 약물 이름
- 복용량
- 계산 방식

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 약물 이름 입력 시 2차원 리스트(데이터베이스)에 존재하는지 확인 (반복문 사용)
- 약물이 없을 시 사용자의 동의를 구하고 필요한 정보를 입력 받아 2차원 리스트에 추가
- while 반복문을 통해 시간이 1시간씩 흐를 때마다 반감기 공식을 적용하여 약물 농도 계산. 이때 현재 농도가 MEC 이상이면 '효과 유지', 미만이면 '효과 소멸'로 조건문 판정을 내리고 결과를 또 다른 2차원 리스트(로그)에 누적

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 시간별 시뮬레이션 표: 0시부터 약효가 소멸되거나 관찰이 끝날 때까지의 [시간, 체내 농도, 약효 상태]를 깔끔한 표 형태로 출력합니다.
- 분석 리포트: 해당 약물의 총 효과 유지 시간(골든타임)을 안내하고, 안전한 추가 복용 타이밍에 대한 권고 메시지를 출력합니다.

## 4. 2차원 리스트 설계

내 프로그램에서 사용할 2차원 리스트의 이름: drug_database / 약물 데이터베이스

### 한 행의 의미
데이터베이스에 등록된 '약물 1종류에 대한 고유한 의약학적 특성'을 의미

### 각 열의 의미

| 열 번호 | 의미 | 예시 |
|---:|---|---|
| 0번 열 | 약물의 이름 | 타이레놀 |
| 1번 열 | 반감기 | 3 |
| 2번 열 | 최소 유효 농도(mec) | 3.0 |


### 2차원 리스트 이름: hourly_log

### 한 행의 의미:
시뮬레이션이 진행되는 동안 매 1시간마다 계산되어 기록되는 '특정 시간대의 체내 약물 잔류 상태 및 판정 결과 데이터'를 의미

### 각 열의 의미:

| 열 번호 | 의미 | 실제 예시 |
|---:|---|---|
| 0번 열 | 복용 후 경과 시간 (정수형, 시간 단위) | 0 |
| 1번 열 | 해당 시간의 체내 잔류 수치 (실수형, mg 또는 mg/L) | 10.0 |
| 2번 열 | 해당 시간의 약효 상태 판정 결과 (문자열) | "🟢 효과 유지" (또는 "🔴 효과 소멸") |
| 3번 열 |  |  |

## 5. 함수 설계

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
| find_drug(name) | 데이터베이스에서 입력받은 약물이 있는지 찾고, 있으면 반감기와 MEC를 꺼내온다. | name (약물 이름) | found, half_life, mec |
| simulate_mec(half_life, mec, initial_dose) | 초기 복용량부터 시작해 시간이 흐름에 따라 농도 감소를 계산하고 로그를 쌓는다. | half_life (반감기), mec (최소유효농도), initial_dose (초기양) | hourly_log, effective_time (총 효과 유지 시간) |
| display_result(log_data, effective_time) | 계산된 2차원 리스트 로그 데이터를 화면에 표 형태로 예쁘게 출력한다. | log_data (2차원 로그 리스트), effective_time (유지 시간) | 화면에 결과값 출력 |

## 6. 사용할 제어구조

### 조건문을 사용할 부분

내 계획:
1. 약물 검색 판정: 사용자가 입력한 약물이 데이터베이스에 존재하는지 확인할 때 (if found:)

2. 새 약물 추가 여부 판정: 없는 약물일 때 사용자가 추가하겠다고 Y를 입력했는지 N을 입력했는지 확인할 때 (if choice == 'Y':)

3. 약효 상태 판정: 시뮬레이션 중 매 시간 현재 약물 농도가 최소 유효 농도(MEC)보다 높은지 낮은지 비교할 때 (if current_amount >= mec:)


### 반복문을 사용할 부분

예) 2차원 리스트의 모든 항목을 하나씩 검사한다.

내 계획:
1. 데이터베이스 탐색: drug_database 내부를 처음부터 끝까지 돌며 사용자가 입력한 약물 이름과 일치하는 행이 있는지 찾을 때 (for drug in drug_database:)

2. 시간 경과 시뮬레이션: 약물 농도가 완전히 사라지거나 지정된 시간(24시간)이 끝날 때까지 1시간씩 증가시키며 농도를 계산할 때 (while current_amount > 1:)

3. 표 형태 출력: 완성된 2차원 리스트(hourly_log)에서 한 행씩 데이터를 꺼내와 화면에 정렬하여 출력할 때 (for row in log_data:)
